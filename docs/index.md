# go-simd documentation

**Pure-Go SIMD drop-ins for the Go standard library** — byte-identical fast
paths for `encoding/base64`, `encoding/base32`, `encoding/hex`, `unicode/utf8`,
`hash/adler32`, `strconv`, and the SIMD-amenable primitives under them.

Every hot loop is **real SIMD assembly generated by
[go-asmgen](https://github.com/go-asmgen/asmgen)** across **amd64, arm64,
loong64 and riscv64** — on plain `go build`, `CGO_ENABLED=0`, stable Go, no
`GOEXPERIMENT`. Each package is a drop-in: the same signatures and
**byte-identical output** (and, where applicable, byte-identical *errors*) as
the stdlib package it accelerates, with the short tail delegated back to the
standard library so results match exactly.

## The honesty policy

These numbers are benchmarked, not hand-waved. Every page reports the **honest
headline** — the wins **and** the parities and the cases where SIMD genuinely
does not help. A multi-table scalar loop that beats the vector unit
([histogram](repos/histogram.md)), or a checksum that lands ~7% behind the best
competitor ([adler32](repos/adler32.md)), is reported as such. The credibility
is the honesty.

## Stdlib drop-ins

| Package | Accelerates | Honest headline |
| --- | --- | --- |
| [`base64`](repos/base64.md) | `encoding/base64` | encode ~17× stdlib; **beats `emmansun/base64` ~5–6%** |
| [`base32`](repos/base32.md) | `encoding/base32` | encode **~7.9× stdlib** (AVX2); no prior pure-Go SIMD base32 |
| [`hex`](repos/hex.md) | `encoding/hex` | **beats `tmthrgd/go-hex` both ways** — encode 20.4×, decode 6.24× stdlib |
| [`utf8`](repos/utf8.md) | `unicode/utf8` | `Valid` **~19× stdlib**, edges `stuartcarnie/go-simd` ~3.5% |
| [`adler32`](repos/adler32.md) | `hash/adler32` | ~12–14× stdlib, but **~7% behind `mhr3/adler32-simd` — honest** |
| [`strconv`](repos/strconv.md) | `strconv` (base-10 parse) | **modest, honest 1.13–1.37×**; never regresses |

## Primitives

| Package | Accelerates | Honest headline |
| --- | --- | --- |
| [`matchlen`](repos/matchlen.md) | LZ match-finder | **~10× scalar** (NEON); real SIMD on all 4 arches |
| [`bitpack`](repos/bitpack.md) | FastPFOR / simdcomp | **~21–32× scalar** — byte-exact vs Lemire's simdcomp |
| [`popcount`](repos/popcount.md) | Hamming weight over `[]byte` | **size-dependent**: ~9–12× in-cache, memory-bound out-of-cache |
| [`histogram`](repos/histogram.md) | byte-value histogram | **multi-table scalar — SIMD doesn't help** (no scatter) |
